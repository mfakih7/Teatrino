# Teatrino Nursery Management System

A modern nursery management platform with a multilingual public website and a powerful Filament admin panel. Teatrino helps nurseries manage children, parents, payments, and weekly reports while presenting a warm, colorful brand online.

## Features

### Public Website
- Multilingual public site in **English**, **Arabic**, and **French**
- Localized CMS content with intelligent fallback when translations are missing
- Home page with hero section, feature cards, and call-to-action buttons
- Articles, portfolio, about, and contact pages
- Responsive Tailwind CSS design with RTL support for Arabic
- Optimized image delivery (thumbnail, optimized, WebP where supported)
- WhatsApp contact links from the public site

### Admin Panel (Filament)
- Dashboard with nursery, payment, and weekly report statistics
- **Website Content** management: site settings, home contents, feature cards, about page, portfolio, articles
- **Nursery Management**: parents, children, payments, weekly child reports
- System settings: admin account, logo, favicon, social links, WhatsApp defaults
- Media uploads with automatic image variant generation
- PDF generation for weekly child reports (DomPDF)
- WhatsApp reminder/parent actions for payments and reports (standard `wa.me` links)

## Local Installation

### Prerequisites
- PHP 8.3+
- Composer 2
- MySQL 8+
- Node.js 18+ and npm
- PHP extensions: `pdo_mysql`, `mbstring`, `openssl`, `tokenizer`, `xml`, `ctype`, `json`, `bcmath`, `fileinfo`, `gd`, `dom`, `libxml`

### Steps

```bash
# Clone the repository
git clone https://github.com/USERNAME/teatrino-nursery-management.git teatrino
cd teatrino

# Install PHP dependencies
composer install

# Environment setup
cp .env.example .env
php artisan key:generate

# Configure your database in .env, then run migrations and seeders
php artisan migrate
php artisan db:seed

# Storage symlink (required for uploaded images)
php artisan storage:link

# Frontend assets
npm install
npm run build

# Filament component cache (recommended after install)
php artisan filament:cache-components
```

### Subdirectory setup (optional)

If the app runs in a subdirectory (e.g. `http://localhost/teatrino`), set:

```env
APP_URL=http://localhost/teatrino
```

The project includes a root `.htaccess` rewrite so URLs do not require `/public` in the path.

### Default admin login

After running `php artisan db:seed`, sign in with:

| Field | Value |
|-------|-------|
| Email | `admin@example.com` |
| Password | `password` |

> **Important:** Change the default admin credentials immediately after first login via **System → System Settings**.

## WhatsApp Notes

Teatrino uses **standard WhatsApp deep links** (`https://wa.me/...`) — there is **no WhatsApp Business API** integration.

- Public contact buttons open WhatsApp with a pre-filled message
- Admin payment reminders open WhatsApp with a reminder message for the parent
- Weekly report sharing opens WhatsApp for the parent, but **PDFs must be attached manually** by the staff member after the chat opens

Configure the default number and message in **Website Content → Site Settings** or **System → System Settings**.

## Image Optimization

Uploaded images are processed automatically and stored in `storage/app/public/media/`:

| Variant | Purpose |
|---------|---------|
| **Original** | Full uploaded file preserved |
| **Thumbnail** | Card/grid previews (~400px) |
| **Optimized** | Main display / modal images (~1920px) |
| **WebP** | Optional lighter format when `MEDIA_GENERATE_WEBP=true` |

Public pages use:
- **Thumbnail** on article and portfolio cards
- **Optimized** in content modals and hero sections
- Native browser **lazy loading** for performance

Ensure `php artisan storage:link` has been run so `/storage/...` URLs resolve correctly.
